# Polar Pattern Generator

Интерактивный генератор симметричных узоров на Jack.

При каждом запуске и по нажатию пробела программа рисует новый уникальный узор - полярную спираль с 4- или 8-кратной симметрией.
Узор строится точка за точкой, анимация видна в реальном времени. Можно менять скорость прямо во время отрисовки и инвертировать цвета (ч/б, б/ч).

## Управление

| Клавиша | Действие |
|---------|----------|
| `Space` | Новый узор |
| `I` | Инверсия цвета (белое на чёрном / чёрное на белом) |
| `Q` | Быстрее |
| `W` | Медленнее |

## Структура проекта

```
src/
  Main.jack  - точка входа, главный цикл, вывод статуса
  Pattern.jack  - параметры узора, генерация координат
  Renderer.jack  - отрисовка, симметрия, управление скоростью
vm/
  *.vm  - скомпилированный код + файлы стандартной OS
```

## Интересные решения

### Полярные координаты без float и тригонометрии
[`Pattern.jack`, методы `getX` и `getY`](https://github.com/SnapD3v/Jack_Patterns/blob/756ec98a4eaa6cd2aaf436205a13cb27b589361d/src/Pattern.jack#L30-L64)
В Jack нет чисел с плавающей точкой и тригонометрических функций.
Вместо `sin`/`cos` использована кусочно-линейная аппроксимация через `if`-ветки -
угол разбивается на четыре квадранта, в каждом считается линейное приближение.
Это даёт непрерывную полярную спираль с нарастающим радиусом (`r = t / 5`)
и параметрически управляемым углом (`angle = t * paramA`).

### Симметрия одним методом
[`Renderer.jack`, метод `drawSymmetric`](https://github.com/SnapD3v/Jack_Patterns/blob/756ec98a4eaa6cd2aaf436205a13cb27b589361d/src/Renderer.jack#L40-L53)

Одна точка `(x, y)` автоматически отражается по всем осям симметрии.
При 4-кратной симметрии из одной точки получается 4, при 8-кратной - 8.
Переключение между режимами - один `if` по полю `symmetry`.

### Коды возврата вместо флагов
[`Renderer.jack`, метод `render`](https://github.com/SnapD3v/Jack_Patterns/blob/756ec98a4eaa6cd2aaf436205a13cb27b589361d/src/Renderer.jack#L64-L97)

В Jack нет исключений и нет способа передать несколько значений из метода.
`render()` возвращает целое число-код: `0` - анимация завершена, `1` - нажат пробел,
`2` - нажата инверсия. `Main` реагирует на каждый код по-своему.
Это чище чем глобальные флаги и не требует дополнительных полей.

### Delay как поле Renderer
[`Renderer.jack`](https://github.com/SnapD3v/Jack_Patterns/blob/756ec98a4eaa6cd2aaf436205a13cb27b589361d/src/Renderer.jack#L5)

Скорость анимации хранится как поле `Renderer`, а не передаётся аргументом в `render()`.
Это позволяет менять `delay` прямо внутри цикла отрисовки и сразу обновлять
текст на экране - без передачи значения обратно в `Main`.

### Псевдослучайная генерация без random()
[`Main.jack`](https://github.com/SnapD3v/Jack_Patterns/blob/756ec98a4eaa6cd2aaf436205a13cb27b589361d/src/Main.jack#L36), [`Pattern.jack`](https://github.com/SnapD3v/Jack_Patterns/blob/756ec98a4eaa6cd2aaf436205a13cb27b589361d/src/Pattern.jack#L18-L25)

В стандартной Jack OS нет генератора случайных чисел. Вместо этого `seed`
увеличивается на 17 при каждом новом узоре, а параметры извлекаются через
битовые операции `&` и `/` - разные биты seed дают независимые параметры
`paramA`, `paramB`, `steps` и `symmetry`. Шаг 17 выбран как простое число,
чтобы последовательность дольше не повторялась.
